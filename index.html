Como funciona
o Base2 Score

Modelo de avaliação

O Base2 Score v4 avalia maturidade em qualidade de software em 6 temas com 37 perguntas. O resultado principal é o radar de maturidade por tema — cada tema recebe um nível independente (Inicial → Avançado) sem agregação forçada. O score numérico (0–100) é referência secundária para acompanhamento histórico.

Os pesos são derivados automaticamente das respostas sobre o contexto de risco: frequência de mudanças, criticidade operacional, complexidade de integração e jornadas de usuário. O relatório inclui ainda uma seção de sugestões de melhoria gerada a partir das respostas com menor score.

Os 6 temas

Testes de Unidade — proteção no nível de código, feedback rápido.

Testes de Integração — cobertura das interações entre componentes.

Testes de Sistema — regressão funcional e pós-deploy.

End-to-End — validação estratégica de jornadas críticas.

Qualidade de Engenharia — quality gates, code review, métricas.

Qualidade Operacional — incidentes, segurança, observabilidade.

Níveis de maturidade

0–39%Inicial — práticas inexistentes. Alto risco operacional.
40–59%Em desenvolvimento — algumas práticas, sem consistência.
60–79%Intermediário — práticas consolidadas, gaps pontuais.
80–100%Avançado — qualidade integrada ao processo.
